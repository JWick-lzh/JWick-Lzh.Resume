## 0. 诚实性铁律（双轨数据口径）

延续原站 `_brief.md` 的诚实原则，数据分两类：

- **成果指标（A 类）**：对外声称的真实战绩，**只用 cc 简历里可核实的数字，不新增不夸大**。
  - 超级麦吉：数万注册用户（对外统一用「2 万+」）、近千万年化收入、Agentlang 首月登 GitHub Trending 全球前四、全网数百万关注；GitHub 4.9k★（线上公开可查）。
  - 效率：投资报告 2 天→数十分钟、会议纪要 5 倍、审批 1 天→1 小时。
  - 连锁 SaaS：300+ 城市、10+ 品牌、单店效率 +40%、补货准确率 85%+、缺货损失 -30%。
  - 智慧客服 6.0：首年 4000 万+、越秀/龙光标杆、工单响应 -60%。
- **演示数据（B 类）**：原型/demo 里为让界面跑起来的示意数据，**可 mock，但保证整套数据自洽闭环**，并在作品内标注「演示数据」。

## 4. 双向单点跳转（核心需求）

- **项目 → 作品**：项目卡点击 → 右侧抽屉详情（iframe 内嵌 / 截图 / 架构图 / 方案，按作品类型切换）。URL：`#work/<id>`，可分享深链。
- **能力 → 作品**：能力标签点击 → 作品区高亮匹配。URL：`#cap/<id>`。
- 深链在页面加载时解析并自动定位（DOMContentLoaded + hashchange）。

## 5. 技术方案（落到现有 React 站）

- 现状：`林灶辉-简历网站` = React 18 + Vite + Tailwind + shadcn + GSAP，SPA，锚点滚动。
- 改动（增量，不推倒视觉骨架）：
  1. **数据层**：新建 `src/data/portfolio.ts`，统一定义 works（含 `tags: capId[]`、`tier`、`media` 类型、`metrics`、`achievements`、`links`）与 caps；成果指标从 cc 简历对齐。
  2. **Skills.tsx**：改为「能力矩阵」——渲染 11 条能力，点击驱动筛选状态（React state / context），删百分比。
  3. **Works.tsx**：按 Tier 分组渲染卡片；接入筛选高亮；卡片点击开抽屉。
  4. **新增 `WorkDrawer.tsx`**：抽屉组件，按 `media.type` 渲染（iframe / shot / arch / plan / run）。
  5. **路由/深链**：轻量 hash 路由（`#work/*`、`#cap/*`），无需引入 router 库。
  6. **静态资源**：原型 HTML 拷进 `public/prototypes/`，截图/架构图进 `public/portfolio/`，iframe/img 引用 `/prototypes/...`、`/portfolio/...`。
  7. **Hero/About**：更新为新叙事主线与四主打；stat 芯片用 A 类真实指标。
- **可运行项目（采购Cloud/AI-Order）**：优先本地可跑 + 补 2–3 分钟录屏；作品详情放方案/架构 + 录屏占位。
- 双语：本期以中文为主，英文留 TODO（不阻塞）。

## 6. 验收标准
